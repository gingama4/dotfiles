# For Copilot: Please write a git commit message for the following changes.
# Gitコミットメッセージを以下の変更に対して作成してください。
# コミットメッセージは日本語で記載し、"Template Rule"に従ってください。
# ファイル下部の変更内容を参照してください。
#
# Subject:

# Description:

# ============================
# Template Rule:
# 1. コミットメッセージは日本語で記載してください。
# 2. コミットメッセージは変更内容を簡潔に要約してください。
# 3. コミットメッセージはConventional Commitsの形式に従ってください。
# 4. コミットメッセージは説明文スタイル(業務文体)で記載してください。
#
# 例)
# feat: 個数を指定してカートに追加できる機能を追加
# ============================
#
# Conventional Commits:
# 以下のフォーマットで記載してください。
# type(scope?): subject
#
# ただし、scopeについては省略可能です。
#
# typeは以下のリストに該当するものを上から選択してください。
# - revert
#   - 特定のコミットの変更を取り消す場合
#   - 基本的にはgit revertの結果として生成されるコミットメッセージの使用を想定
# - build
#   - ビルドプロセスに関係する変更
#   - 例: ビルドツールの設定変更、依存関係
#   - ワークフローファイルは含めない
# - test
#   - テストコードの追加や修正
#   - ユニットテストコードについては要検討、本体コードとセットで変更される場合はfeatやfixを使用を推奨
# - ci
#   - CI/CD設定の変更
#   - ワークフローファイルの追加や修正
# - style
#   - フォーマッター・リンターによる機械的な変更
#   - コードの動作に影響しない変更
#   - 基本的に機械的な変更のみに限定する
# - refactor
#   - 外部仕様を変えない内部構造の改善
#   - プライベート変数の変数名変更
#   - 機械的ではないコードフォーマット
# - perf
#   - パフォーマンスを向上させるコード変更
#   - 明らかに低速なアルゴリズムの改善など
# - docs
#   - ドキュメントの変更
#   - READMEやコード内コメントの修正・追加
#   - タイポの修正
# - fix
#   - バグ修正
#   - 可能な限りバグフィックスに限定する
# - feat
#   - 新機能の追加
#   - 既存機能の大幅な改善
#   - ユーザーにとって新しい価値を提供する変更
# - chore
#   - その他の変更
#   - 複数の軽微な修正が混在する場合
#     - Typo+コメント修正+フォーマット調整など
#   - できるだけ他のtypeを使用することを推奨
# ============================
#
